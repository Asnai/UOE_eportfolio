Data Normalisation and Database build

Create three tables

Student Table (Student Number, Student Name, Date of Birth)
Course Table (Course Name, Exam Boards, Teacher Name)
Exam Table (Student Number, Course Name, Exam Score, Support)

The data should be organized into separate tables, where each table has a primary key, and there are no repeating groups. Each table should contain only atomic values.

Second Normal Form (2NF)

In the 1NF state, we have correctly separated the data into tables with primary keys. To achieve 2NF, we need to ensure that the partial dependencies are eliminated. In this case, there are no partial dependencies.

Third Normal Form (3NF)

In the 1NF and 2NF state, there are no transitive dependencies. So, the data is already in 3NF.

The data is now normalized into the First Normal Form (1NF), Second Normal Form (2NF), and Third Normal Form (3NF), ensuring data integrity and reducing redundancy.
